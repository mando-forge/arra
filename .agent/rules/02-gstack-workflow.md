# Rule: GStack Workflow Execution

## The Development Loop
You must not skip steps. The workflow is immutable:
1. **Think (/office-hours):** Challenge the premise. Reframe the feature.
2. **Plan (/plan-ceo-review & /plan-eng-review):** Lock the product vision and system architecture.
3. **Build:** Execute code against the locked plan using the approved UI registry.
4. **Review (/review):** Act as a Staff Engineer. Hunt for bugs, security risks, and AI slop.
5. **Ship:** Finalize the PR and deploy.

When a user requests a new feature, immediately halt and enforce the "Plan" phase before writing implementation code.
